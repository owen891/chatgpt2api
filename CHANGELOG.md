# Changelog

## Unreleased

+ [修复] 图片和缩略图地址显式支持 HEAD 请求，避免外部平台预检图片时被前端兜底页错误响应为 HTML。
+ [修复] 自动重登增加 2 个并发上限并去重，避免批量 refresh token 失效时触发登录请求风暴。
+ [安全] debug 账号日志不再写入邮箱、用户 ID 或账号 ID；示例配置默认关闭 debug 和自动重登。
+ [运维] 增加 root-only 的本地 SQLite 快照定时任务；R2 云备份仍需填写完整的 Cloudflare R2 凭据后启用。
+ [修复] 注册账号必须通过远端账号接口验证后才计为成功；服务端已撤销的 token 会移出账号池并计入注册失败。
+ [修复] 图片上游默认超时由 90 秒提高到 360 秒，覆盖上游内部轮询和跨账号重试耗时，避免上游仍在后台生成时本项目提前报错。
+ [修复] ChatGPT 后台已创建生图会话但本地轮询超时时，保留原会话 ID 并停止换账号或外部回退，避免后台已生成图片却被覆盖为“无可用生图账号”，同时允许前端继续等待原结果。

## 1.1.1 - 2026-07-17

+ [修复] 注册流程可识别 passwordless OTP 的 HTTP 429 限流，并按配置停止领取新任务、进入冷却后自动恢复。
+ [优化] 注册任务启动时先使用单线程探测出口，探测成功后再恢复配置并发，避免不可用出口瞬间消耗整批邮箱。
+ [优化] 429 冷却结束后先以单线程验证出口，成功后恢复配置并发。
+ [新增] 注册设置支持配置 429 冷却时间，并明确注册代理留空时继承全局代理。
+ [修复] 用户手动停止注册任务时清理旧的限流原因，运行状态显示为用户手动停止。
+ [测试] 增加 OTP 429 分类、冷却、启动探测、恢复探测和手动停止的回归测试。

## 1.1.0 - 2026-07-16

+ [新增] 注册流程阶段诊断：记录每次尝试的阶段顺序、耗时、失败阶段与执行流类型。
+ [新增] 注册服务端诊断聚合：新增漏斗统计、渠道表现、失败类型分布和最近失败列表。
+ [新增] 注册页诊断面板：前端可直接查看阶段漏斗、渠道表现与最近失败记录。
+ [重构] 注册页改为页面壳 + runtime hook + panels 结构，拆分设置面板、运行面板、诊断面板、来源面板和 provider 卡片。
+ [测试] 为注册诊断聚合补充回归测试，并完成前端 TypeScript 校验。
+ [修复] 账号池凭证失效、配额耗尽或暂不可用时，生图请求可继续回退到已配置的外部上游。
+ [修复] OAuth 刷新错误区分永久失效与临时故障，减少无效账号重复参与调度。

## 1.0.0 - 2026-07-14

+ [新增] 二次开发版本基线，仓库地址切换为 `owen891/chatgpt2api`。
+ [新增] 生图上游管理：Base URL、API Key、模型映射、默认渠道、超时、代理、连通性测试。
+ [新增] `/v1/images/generations` 与 `/v1/images/edits` 支持账号池优先、上游故障回退。
+ [新增] 多上游按优先级故障切换，支持上游返回图片 URL 后下载保存到本地图库。
+ [新增] 独立注册机页面与菜单入口，支持常驻监控、额度不足自动补池、实时日志与运行指标。
+ [新增] 生图任务支持单张停止、停止剩余生成与停止后重新生成。
+ [调整] 前端 GitHub 链接、版本展示与版本检查改为指向二改仓库。
+ [安全] 增加 `config.example.json`，本地 `config.json`、`data/`、`logs/`、`.env` 等运行期敏感数据不再作为提交内容。
